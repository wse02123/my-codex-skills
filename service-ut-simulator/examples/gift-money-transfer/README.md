# 경조금 이체 후 메모지 등록 샘플

이 폴더는 `service-ut-simulator` 스킬을 처음 써보는 사람이 바로 테스트해볼 수 있는 예시 자료입니다.

## 사용 방법

1. Codex에서 `service-ut-simulator` 스킬을 실행합니다.
2. `screen-flow.svg` 이미지를 화면 자료로 제공합니다.
3. 아래 필수정보를 함께 입력합니다.

## 필수정보

1. 화면 프로세스 이미지 또는 목업  
   - `screen-flow.svg`

2. 화면 순서  
   - 이체 완료 화면 → 경조금 카테고리 선택 바텀시트 → 경조사메모지 등록 완료 상태

3. 테스트 사용자 맥락  
   - 기존 사용자

4. 검증할 기능과 사용자가 완료해야 하는 핵심 행동  
   - 이체를 한 사용자가 경조금을 이체했다면, 경조금 카테고리를 지정하여 경조사메모지에 등록할까?

5. 현재 예상 타겟  
   - 30대 결혼식을 자주 참가하는 사람
   - 50대 장례식을 자주 참가하는 사람

## 선택정보 예시

아래 정보는 없어도 실행할 수 있지만, 함께 넣으면 결과가 더 선명해집니다.

### 서비스/기능 맥락

- 서비스명: 경조사메모지
- 기능 설명: 이체 완료 후 축의금/부의금 등 경조금 카테고리를 선택해 경조사메모지에 바로 등록하는 기능
- 사용자가 얻어야 하는 핵심 가치: 경조금 이체 내역을 따로 기억하거나 수기로 적지 않아도 자동으로 정리할 수 있음
- 검증하고 싶은 가설: 이체 직후에는 경조금 맥락이 가장 선명하므로, 이 시점에 등록 제안을 하면 사용자가 자연스럽게 등록할 것이다.
- 특히 걱정되는 UX 지점: 사용자가 `경조사메모지`의 의미를 이해하지 못하거나, 이체 완료 후 빨리 화면을 닫고 싶어 등록을 건너뛸 수 있음

### 사용자 맥락

- 기존 사용자라면: 이미 이체 기능을 자주 사용하고, 경조사 내역을 따로 관리해본 경험이 있을 수 있음
- 기존에 같은 일을 처리하던 방식: 이체 후 메신저, 메모앱, 엑셀, 기억에 의존해 경조금 내역을 정리

### 성공 기준

- 성공으로 볼 행동: 사용자가 카테고리를 선택하고 `등록`을 눌러 경조사메모지 등록을 완료함
- 중간에 반드시 거쳐야 하는 단계: 등록 제안 인지 → 카테고리 확인/변경 → 등록 → 등록 완료 상태 확인
- 사용자가 이탈하면 안 되는 중요 지점: 이체 완료 직후 등록 제안 영역, 카테고리 선택 바텀시트

### 제약 조건

- 모바일 앱 기준
- 실제 이체는 이미 완료된 상태
- `계속이체`와 `확인` 버튼이 같은 화면에 함께 존재함
- 테스트 범위는 경조사메모지 등록 완료와 내역 확인 진입까지
